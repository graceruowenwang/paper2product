# Notion 数据库适配方案

> 如果你使用 Notion 作为主力管理工具，以下是推荐的数据库结构。

---

## 数据库设计

### 1. 论文数据库（核心）

**用途**：管理所有论文的流转状态

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | Title | 论文标题 |
| 状态 | Select | inbox / 评估中 / 阅读中 / 写作中 / 已发布 / 归档 |
| 来源 | Select | arXiv / NeurIPS / ICML / HuggingFace / Twitter |
| 产品化评分 | Number | 选题评估得分（/10） |
| 方向 | Multi-select | RAG / Agent / NLP / CV / 推荐 / 多模态 / 安全 |
| 发布日期 | Date | 文章发布日期 |
| 论文链接 | URL | arXiv/会议链接 |
| 代码仓库 | URL | GitHub 链接（如有） |
| 文章链接 | URL | 微信公众号文章链接 |
| 研究笔记 | Relation | 关联到研究笔记数据库 |
| 创建时间 | Created time | 自动记录 |

**视图建议**：
- 看板视图：按状态流转
- 日历视图：按发布日期
- 表格视图：评分排序

### 2. 研究笔记数据库

**用途**：记录每篇论文的深度研究

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | Title | 论文标题 |
| 关联论文 | Relation | 关联到论文数据库 |
| 核心问题 | Text | 一句话 |
| 方法直觉 | Text | 通俗解释 |
| 产品化分析 | Text | 可做产品列表 |
| 实现评估 | Select | 低/中/高 |
| 竞品信息 | Text | 相关项目和产品 |
| 写作角度 | Text | 建议的切入点 |

### 3. 内容日历数据库

**用途**：管理发布排期

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | Title | 文章标题 |
| 类型 | Select | 深度解读 / 周报 / 月度复盘 |
| 计划日期 | Date | 计划发布日 |
| 实际日期 | Date | 实际发布日 |
| 状态 | Select | 计划中 / 写作中 / 审核中 / 已发布 |
| 关联论文 | Relation | 关联到论文数据库 |
| 阅读量 | Number | 发布后 24h |
| 分享数 | Number | |
| 新增关注 | Number | |

**视图建议**：
- 日历视图：按日期查看排期
- 看板视图：按状态管理进度

### 4. 灵感库数据库

**用途**：收集零散的选题灵感和产品想法

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 内容 | Text | 灵感/想法描述 |
| 来源 | Select | 阅读 / 聊天 / 观察 / 竞品分析 |
| 方向 | Multi-select | 同论文数据库 |
| 优先级 | Select | 高/中/低 |
| 是否已用 | Checkbox | |

## Notion 页面结构

```
Paper2Product (主页)
├── 仪表盘（Dashboard）
│   ├── 本周待办（Calendar View）
│   ├── 论文看板（Board View by Status）
│   └── 快捷入口（链接到各子页面）
├── 论文数据库
├── 研究笔记
├── 内容日历
├── 灵感库
├── 模板库
│   ├── 论文解读模板
│   ├── 周报模板
│   ├── 选题评估模板
│   └── 月度复盘模板
├── Prompt 库
│   ├── 提取关键信息
│   ├── 生成初稿
│   ├── 优化标题
│   └── 生成摘要
└── 运营数据
    ├── 月度数据汇总
    └── 复盘记录
```

---

## 与本地文件同步建议

- 本地 Markdown 文件作为备份和版本控制
- Notion 作为日常操作入口
- 重要模板同时维护在两个地方
- 使用 Notion API + Python 脚本可做双向同步（按需）
