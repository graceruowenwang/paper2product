# Paper2Product 论文解读模板

> 每篇文章遵循以下固定结构，确保读者形成阅读预期。

---

## 元数据（发布前填写）

- 论文标题：
- 作者/机构：
- 来源（arXiv/NeurIPS/ICML/ACL/etc）：
- 发布日期：
- 文章发布日期：
- 文章标签：#论文解读 #应用方向 #技术类别

---

## 标题格式

`[论文解读] 一句话产品价值点 | 原论文关键词`

示例：`[论文解读] 让 AI 记住你的使用习惯 -- 基于 Retrieval-Augmented 的个性化对话系统`

---

## 正文结构

### 1. 一句话价值（开篇钩子）

用一句话说清这篇论文的产品价值。读者看完这句就能决定是否继续阅读。

> **如果你在做 [某类产品]，这篇论文给了你一个 [具体能力] 的实现思路。**

### 2. 这篇论文解决什么问题

- **现实痛点**：用产品经理语言描述当前行业/用户面临的痛点（2-3句）
- **现有方案的局限**：一句话概括现有方案的不足
- **论文的核心突破**：一句话说清论文的方法创新

### 3. 核心方法解读（产品经理版）

用类比和图示解释技术方案，避免公式推导。

- **一句话原理**：用日常类比解释核心算法思路
- **关键组件**：列出 2-4 个关键模块，每个用一句话说明功能
- **技术门槛评估**：实现难度（低/中/高）+ 需要的核心能力
- **数据需求**：训练/推理需要什么数据，数据获取难度

### 4. 产品化分析（核心章节）

#### 4.1 直接可做产品

列出可以直接基于此论文能力构建的产品形态：
- 产品名称（假设）
- 目标用户
- 核心功能
- 变现模式
- MVP 周期估算

#### 4.2 增强现有产品

此能力可以嵌入哪些已有产品中，提升什么体验：
- 适用产品类型
- 嵌入点
- 预期提升指标

#### 4.3 护城河分析

- 技术壁垒高低
- 复现难度
- 数据壁垒
- 可能的防御策略

### 5. 竞品与生态

- 已有的类似产品/开源项目
- 论文作者是否有商业化动作
- 相关的开源实现链接（如有）

### 6. 行动建议

按读者角色给出差异化建议：
- **创业团队**：建议做什么、切入点
- **大厂 PM**：如何整合到现有产品线
- **独立开发者**：快速验证的最小方案

### 7. 论文信息卡

| 字段 | 内容 |
|------|------|
| 论文链接 | |
| 代码仓库 | |
| 关键词 | |
| 阅读时长 | |
| 推荐指数 | (1-5) |
| 产品化就绪度 | (研究阶段/可原型/可商用) |

---

## 写作要求

- 全文 2500-4000 字
- 每个章节有明确的小标题
- 用加粗标记关键信息
- 涉及技术概念时，先用类比解释再给术语
- 产品分析部分占全文 40% 以上篇幅
- 不放数学公式，用文字描述直觉
