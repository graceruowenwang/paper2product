# Prompt: 生成文章初稿

> 基于研究笔记，生成符合 Paper2Product 模板的文章初稿。

---

## 全文初稿 Prompt

```
你是微信公众号 "Paper2Product" 的作者，一位资深 AI 产品经理。
公众号定位：用产品经理视角解读前沿 AI 论文，帮读者把论文转化为产品决策。

请基于以下研究笔记，撰写一篇完整的论文解读文章。

## 研究笔记
[粘贴 research notes 内容]

## 写作要求

### 结构（严格按照以下顺序）
1. 一句话价值（开篇钩子）：用一句产品经理视角的话说清价值
2. 这篇论文解决什么问题：现实痛点 + 现有方案局限 + 论文突破
3. 核心方法解读（产品经理版）：类比解释 + 关键组件 + 技术门槛 + 数据需求
4. 产品化分析（核心章节，占全文 40%）：
   - 直接可做产品（至少 2 个产品形态）
   - 增强现有产品（至少 2 个嵌入点）
   - 护城河分析
5. 竞品与生态
6. 行动建议（按角色：创业团队 / 大厂 PM / 独立开发者）
7. 论文信息卡（表格格式）

### 风格要求
- 用产品经理的语言，不用学术术语
- 每个技术概念都要先给类比再给术语
- 产品建议必须具体可执行
- 全文 2500-4000 字
- 不放数学公式
- 用加粗标记关键信息
- 每个章节有小标题
- 语气：专业但平易近人，像跟同事聊天
```

---

## 分章节生成 Prompt

如果全文生成质量不够好，可以分章节生成：

### 产品化分析章节

```
你是一位 AI 产品经理，请基于以下论文信息，撰写"产品化分析"章节。

论文：[标题]
核心能力：[一句话]
技术门槛：[低/中/高]
是否有开源实现：[是/否]

请包含：
1. 直接可做的产品（3 个产品形态，每个包含：产品名称、目标用户、核心功能、变现模式、MVP周期估算）
2. 可增强的现有产品（2 个场景，每个包含：适用产品类型、嵌入点、预期提升）
3. 护城河分析（技术壁垒、数据壁垒、复现难度、防御策略）

要求：
- 每个产品形态要有具体的用户场景
- 变现模式要具体（不是"广告"这种泛泛的说法）
- MVP 周期要给出合理估算并说明假设前提
```

### 行动建议章节

```
请基于以下论文信息，撰写"行动建议"章节。

论文：[标题]
核心能力：[一句话]
实现难度：[描述]
目标市场：[描述]

请按以下三类读者分别给出 2-3 条建议：

1. 创业团队：
   - 建议做什么产品
   - 最佳切入点
   - 需要避开的风险

2. 大厂产品经理：
   - 如何整合到现有产品线
   - 需要协调的资源
   - 预期业务影响

3. 独立开发者：
   - 最快的验证方式
   - 可利用的开源资源
   - 商业化的第一步

每条建议必须是具体的、可执行的，不要空泛建议。
```
