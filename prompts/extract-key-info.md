# Prompt: 提取论文关键信息

> 用于快速提取论文的核心信息，生成结构化笔记。

---

## Prompt

```
你是一位 AI 产品经理，专注于将前沿论文转化为产品洞察。请分析以下论文，提取关键信息。

## 输出格式

### 论文元数据
- 标题（英文原文 + 中文翻译）
- 作者（前 3 位）和机构
- 发布时间和来源

### 核心贡献（严格 3 句话）
1. 问题定义句：[这篇论文解决什么问题]
2. 方法创新句：[用了什么新方法]
3. 结果亮点句：[最关键的实验结果]

### 技术解析
- 核心方法的一句话解释（面向 PM，用类比）
- 与现有方法的关键区别
- 实现这个方法需要的技术栈

### 实验结果解读
- 主实验：[任务] 上 [指标] 提升了 [百分比]
- 最有说服力的消融实验结果
- 在什么条件下方法可能失效

### 数据与资源需求
- 训练数据量和类型
- 推理时的计算需求
- 是否有开源代码/模型

### 一句话产品化判断
[这篇论文最值得做的一个产品方向是什么？为什么？]

---

论文内容：
[粘贴论文摘要或全文]
```

## 使用场景

- inbox 论文的快速评估
- 为选题评估模板提供输入
- 为周报速览提取关键信息

## 使用建议

- 对于长论文，可以分段输入，每次聚焦一个章节
- 先输入摘要 + 结论，获取整体判断，再决定是否需要深入
- 输出可以直接作为 research notes 的基础
