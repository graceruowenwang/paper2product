# 写作流程（含 AI Prompt 链）

## 总体原则

- 人做判断和结构设计，AI 做信息提取和初稿生成
- 每篇文章总耗时控制在 2-3 小时（含阅读研究）
- 先出结构再填内容，不要从第一段开始写

---

## 写作流程（5 步法）

### Step 1：确定角度（15 分钟）

基于 research notes，确定文章的产品化角度：
- 这篇文章的核心产品洞察是什么？
- 目标读者是谁（创业 PM / 大厂 PM / 独立开发者）？
- 最吸引人的标题方向？

### Step 2：生成文章骨架（10 分钟）

用 AI 生成结构化大纲：

```
Prompt：
我正在为微信公众号 "Paper2Product" 撰写一篇论文解读文章。
读者是 AI 产品经理。

论文信息：
- 标题：[xxx]
- 核心贡献：[xxx]
- 产品化角度：[xxx]

请按以下结构生成详细大纲（每个章节列出 2-3 个要点）：
1. 一句话价值（开篇钩子）
2. 这篇论文解决什么问题
3. 核心方法解读（产品经理版）
4. 产品化分析（直接可做 / 增强现有 / 护城河）
5. 竞品与生态
6. 行动建议
7. 论文信息卡

注意：每个要点用产品经理语言描述，避免学术术语。
```

### Step 3：AI 生成初稿（30 分钟）

分章节让 AI 生成初稿，每个章节单独生成，便于控制质量：

```
Prompt（以"产品化分析"章节为例）：
基于以下论文信息，撰写文章的"产品化分析"章节，约 800 字。

论文：[标题]
核心方法：[一句话描述]
关键能力：[列出 2-3 个]

请包含以下内容：
1. 直接可做的产品（至少 2 个产品形态，每个包含：产品名、目标用户、核心功能、变现模式、MVP周期）
2. 可增强的现有产品（至少 2 个嵌入点）
3. 护城河分析（技术壁垒、数据壁垒、复现难度）

风格要求：
- 用产品经理的语言，不用学术术语
- 具体且可执行，不要空泛建议
- 每个产品形态要有用户场景描述
```

### Step 4：人工润色（60 分钟）

- 通读全文，调整逻辑流
- 补充个人观点和判断
- 修改 AI 生成的不准确之处
- 加入类比和案例
- 调整语气，确保统一

### Step 5：标题优化（15 分钟）

生成 5 个候选标题，选择最优：

```
Prompt：
为以下文章生成 5 个微信公众号标题，要求：
1. 突出产品价值点，不是学术概念
2. 控制在 20 字以内
3. 使用数字、疑问句或对比句式
4. 避免标题党但要有吸引力

文章核心内容：[一句话总结]
目标读者：AI 产品经理
```

---

## 时间分配参考

| 步骤 | 时间 | AI 参与 |
|------|------|--------|
| 确定角度 | 15min | 低 |
| 生成骨架 | 10min | 高 |
| 生成初稿 | 30min | 高 |
| 人工润色 | 60min | 低 |
| 标题优化 | 15min | 中 |
| 排版发布 | 30min | 低 |
| **合计** | **~2.5h** | |
