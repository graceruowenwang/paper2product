# 阅读与研究流程

## 目标

每篇论文的阅读研究控制在 1-2 小时内完成，产出可直接用于写作的素材笔记。

---

## 三遍阅读法

### 第一遍：5 分钟，判断值不值得读

- 读标题、摘要、结论
- 看图表和标题
- 决定：跳过 / 进入第二遍 / 直接写（如果非常熟悉领域）

### 第二遍：30 分钟，理解核心贡献

- 读懂 Introduction 和 Method 的核心思路
- 不纠结数学公式，关注 intuition
- 记录：
  - 核心问题定义
  - 方法的一句话总结
  - 关键实验结果
  - 局限性
- 决定：不够产品化则跳过 / 进入第三遍

### 第三遍：60 分钟，深度产品分析

- 读懂所有实验和 ablation study
- 分析：
  - 技术实现的关键细节
  - 计算资源需求
  - 数据需求
  - 与现有方案的对比
- 搜索：
  - 是否有开源实现
  - 是否有竞品
  - 作者团队背景和商业化可能
- 产出一份完整的 research notes

## AI 辅助阅读 Prompt

在阅读过程中，可以用以下 Prompt 加速：

### 快速理解论文核心
```
我是一名词产品经理，正在阅读以下论文。请帮我：
1. 用一句话总结核心贡献
2. 用非技术语言解释核心方法
3. 列出 3 个可能的产品应用场景
4. 评估实现难度（低/中/高）并说明理由

论文内容：[粘贴摘要或关键段落]
```

### 评估实验结果
```
请帮我分析以下论文实验结果的产品化意义：
1. 这个结果在实际应用中意味着什么？
2. 与现有方案相比，提升了多少？提升有实际意义吗？
3. 需要什么条件才能复现这个结果？
4. 在什么条件下这个方法可能失效？

实验结果：[粘贴相关内容]
```
